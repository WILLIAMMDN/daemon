Politica de privacidad de DAEMON

Version vigente: 15 de julio de 2026.

DAEMON es una plataforma educativa con experiencias diferenciadas KIDS y TEENS. Esta politica explica que datos usa la plataforma y como se protegen.

Datos tratados

La cuenta puede incluir nombre, correo, nombre de usuario, nivel educativo, aula, progreso, respuestas, entregas, insignias, canjes y contenido creado dentro de los modulos. Para cuentas KIDS se solicita el correo de una madre, padre o tutor y se conserva cifrado.

Finalidades

Los datos se usan para autenticar la cuenta, prestar las funciones educativas, guardar progreso, permitir el acompanamiento docente, proteger la plataforma y atender solicitudes de privacidad. DAEMON no vende datos personales.

Proveedores de infraestructura

La plataforma utiliza Firebase para identidad y frontend, Render para la API, Supabase para base de datos y archivos, Pusher para eventos en tiempo real y Sentry para diagnostico tecnico con muestreo reducido. Cada proveedor procesa solo la informacion necesaria para su funcion.

KIDS y TEENS

El nivel KIDS exige una declaracion de conocimiento de una madre, padre o tutor. No se almacena una fecha de nacimiento exacta. El nivel seleccionado se usa para separar contenido y experiencia educativa.

Seguridad y conservacion

Las evidencias academicas se guardan en almacenamiento privado mediante enlaces temporales. Los secretos se cifran, las sesiones se protegen y los registros efimeros se eliminan segun plazos definidos. El historial academico no se elimina automaticamente mientras pueda existir una obligacion educativa o legal de conservarlo.

Derechos y solicitudes

Una persona autenticada puede exportar sus datos y registrar una solicitud de eliminacion. La eliminacion se revisa antes de ejecutarse para proteger la cuenta, preservar obligaciones academicas aplicables y evitar solicitudes fraudulentas.

Cambios

Cuando cambie materialmente esta politica, DAEMON publicara una nueva version y solicitara una nueva aceptacion cuando corresponda.
